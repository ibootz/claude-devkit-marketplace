# Heal Skill - 技能修复工具

在技能执行过程中发现问题时，使用此技能更新 SKILL.md 和相关文件。

## 目标

分析对话历史，检测正在运行的技能，反思出错原因，提出具体修复方案，获得用户批准后应用更改并可选择性提交。

## 工作流程

### 1. 检测技能

从对话上下文中识别技能：

- 查找技能调用消息
- 检查最近引用的 SKILL.md
- 检查当前任务上下文

设置：`SKILL_NAME=[技能名称]` 和 `SKILL_DIR=./skills/$SKILL_NAME`

如果不清楚，询问用户。

### 2. 反思与分析

如果提供了 $ARGUMENTS，则专注于该问题，否则分析更广泛的上下文。

确定：

- **问题所在**：引用 SKILL.md 中不正确的具体部分
- **发现方法**：上下文、错误消息、试错、文档查找
- **根本原因**：过时的 API、错误参数、错误端点、缺失上下文
- **影响范围**：单个部分还是多个？相关文件是否受影响？
- **建议修复**：哪些文件、哪些部分、每个的前后对比

### 3. 扫描受影响的文件

```bash
ls -la $SKILL_DIR/
ls -la $SKILL_DIR/references/ 2>/dev/null
ls -la $SKILL_DIR/scripts/ 2>/dev/null
```

### 4. 展示建议更改

以此格式展示更改：

```
**正在修复的技能：** [技能名称]
**发现的问题：** [1-2 句总结]
**根本原因：** [简要说明]

**要修改的文件：**
- [ ] SKILL.md
- [ ] references/[文件].md
- [ ] scripts/[文件].py

**建议更改：**

### 更改 1: SKILL.md - [部分名称]
**位置：** SKILL.md 第 [X] 行

**当前（错误）：**
```

[当前文件的确切文本]

```

**修正后：**
```

[新文本]

```

**原因：** [为什么这能修复问题]

[对所有文件的每个更改重复]

**影响评估：**
- 影响：[身份验证/API 端点/参数/示例/等]

**验证：**
这些更改将防止：[导致此问题的具体错误]
```

### 5. 请求批准

```
是否应用这些更改？

1. 是，应用并提交所有更改
2. 应用但不提交（让我先审查）
3. 修改更改（我会提供反馈）
4. 取消（不做更改）

选择 (1-4)：
```

**等待用户响应。未经批准不要继续。**

### 6. 应用更改

仅在批准后（选项 1 或 2）：

1. 对所有文件的每个修正使用 Edit 工具
2. 读回修改的部分以验证
3. 如果选项 1，使用结构化消息提交，显示修复内容
4. 用文件列表确认完成

## 成功标准

- 从对话上下文正确检测技能
- 识别所有不正确的部分并提供前后对比
- 应用前获得用户批准
- 在 SKILL.md 和相关文件中应用所有编辑
- 通过读回验证更改
- 如果用户选择选项 1，则创建提交
- 用文件列表确认完成

## 验证

完成前：

- 读回每个修改的部分以确认应用了更改
- 确保跨文件一致性（SKILL.md 示例与 references/ 匹配）
- 如果选择了选项 1，验证创建了 git 提交
- 检查没有修改意外文件
