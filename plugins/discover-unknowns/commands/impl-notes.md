现在进入实现，全程维护**实现笔记（Implementation Notes）**：$ARGUMENTS

工作方式：
1. 在仓库根目录维护一个临时文件 `implementation-notes.md`（若我更想要 HTML 则用 `.html`）。
2. 按已确认的计划/规格推进实现。
3. 每当撞上一个**迫使你偏离计划的边界情况**：**选择保守的方案**，把「遇到了什么 / 为什么偏离 / 选了哪个方案 / 放弃了什么」记录到笔记的 **`## Deviations`** 小节下，然后**继续推进**，不要停下来反复追问。
4. 同时在笔记里记录关键决策、假设、以及留给下一次尝试的经验。
5. 完成后向我汇报：做了什么、`Deviations` 里有哪些偏离需要我复核。

提醒：无论计划多充分，总有"未知的未知"潜伏——笔记的目的就是让这些发现可追溯、可复用。
