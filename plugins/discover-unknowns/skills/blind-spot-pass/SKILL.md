# 盲点扫描（Blind Spot Pass）

对用户给出的任务/领域（随调用传入的参数或当前对话上下文）做一次盲点扫描。

## 执行步骤

1. 先复述你从用户背景中理解到的"用户是谁、已知什么、不熟悉什么"，据此决定如何协作。背景不足以判断时，先问 1-2 个最关键的问题。
2. 系统性列出这个领域里用户最可能存在的**未知的未知（unknown unknowns）**：
   - 不知道该问的问题
   - "好"的标准长什么样
   - 历史上做过哪些相关工作
   - 有哪些常见的坑
3. 逐条**讲解**清楚，让用户能理解，并据此**更好地向你提问**。
4. 最后给出：接下来应优先澄清哪 2-3 个未知，以及对应的追问方向。

## 边界

- 不要现在就开始实现；本模式只为暴露和讲解未知。
- 讲解要基于对代码库/领域的实际检索与事实，不要泛泛而谈行业常识。

## 原文示例 prompt（供理解意图）

- "我正在为代码库添加一个新的认证提供方，但我对这个代码库里的认证模块一无所知。你能做一次盲点扫描，帮我理清相关的未知的未知，并帮助我更好地向你提问吗？"
- "我不知道什么是调色（color grading），但我需要给这段视频调色。你能教我理解自己关于调色的未知的未知，好让我能更好地向你提问吗？"
