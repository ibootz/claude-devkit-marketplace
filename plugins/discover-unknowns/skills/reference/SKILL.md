# 参考（Reference）

用户给出了想要效果的参考（随调用传入的参数：参考的路径/库/组件 + 想要它的哪部分行为）。

## 执行步骤

1. 优先把它当**源代码参考**来读：即使是另一种语言写的，源码提供的标记与结构细节也远胜截图或文字描述。
2. 先**读懂并向用户复述**这个参考里用户真正想要的是哪部分语义/行为/结构，确认双方对"要复刻什么"理解一致。
3. 指出参考与当前代码库/技术栈之间的差异，以及复刻时需要的适配取舍。
4. 确认一致后，在当前项目里**重新实现相同的语义**（而非照搬无关细节）。

## 边界

- 参考不够充分以定位用户想要的东西时，先说明还需要什么参考（哪个文件夹、哪个组件、哪段行为），不要猜。

## 原文示例 prompt（供理解意图）

- "vendor/rate-limiter 里的这个 Rust crate 实现了我想要的退避（backoff）行为。读一读它，并在我们的 TypeScript API 客户端里重新实现相同的语义。"
