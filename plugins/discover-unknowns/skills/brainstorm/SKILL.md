# 头脑风暴与原型（Brainstorms & Prototypes）

围绕用户给出的目标（随调用传入的参数或当前对话上下文）做头脑风暴与原型。

## 优先委派 superpowers（若已安装）

先检查当前可用技能列表中是否存在 `superpowers:brainstorming`（用户已安装 [superpowers](https://github.com/obra/superpowers) 插件）：

- **存在** → 通过 Skill 工具调用 `superpowers:brainstorming`，把用户的目标原样传入，按其流程执行（自然对话式收敛意图 → 设计 → 用户确认）。本文件下方的"原型"原则仍作为补充约束生效：可视化产物用假数据单 HTML 原型、多方向供反应。
- **不存在** → 按下方自定义指令执行。

## 自定义指令（fallback）

### 两种形态（按情况选择，或都做）

- **多方向探索**：为同一产物给出多个**风格迥异**的方向（默认 3-4 个）供用户反应。若是可视化产物（UI/仪表盘/布局），产出**单独的 HTML 文件、用假数据**呈现——不接后端、不维护多余状态。
- **范围头脑风暴**：搜索相关代码库/上下文，头脑风暴多个可介入的方案，**从最省成本到最有雄心排列**，由用户挑选有共鸣的。

### 原则

1. 价值在于**尽早、廉价地暴露"未知的已知"**（用户只有看到才知道如何定义的标准），避免拖到实现阶段才浮现——那时功能/规格的小改动会造成代码实现的巨大差异，且回退更难。
2. 不接通真实系统，不做超出"可反应原型"所需的工作。
3. 主动指出你发现的高价值但用户可能错过的方向，并提醒范围是否太窄或太宽。
4. 产出后**等用户反应**，不要直接进入实现。

## 原文示例 prompt（供理解意图）

- "我想为这些数据做一个仪表盘，但我没有视觉审美，也不知道有哪些可能性。给我做一个 HTML 页面，包含 4 个风格迥异的设计方向，好让我做出反应。"
- "在接通任何东西之前，用假数据做一个单独的 HTML 文件，模拟新的编辑器工具栏。我想在你动真正的应用之前先对布局做出反应。"
- "这是我大致的问题：用户在引导之后流失。搜索代码库，头脑风暴出 10 个我们可以介入的地方，从最省成本到最有雄心排列。我会告诉你哪些让我有共鸣。"
