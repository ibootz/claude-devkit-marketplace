# 访谈（Interview）

就用户给出的任务（随调用传入的参数或当前对话上下文）对用户进行访谈。

## 优先委派 superpowers（若已安装）

先检查当前可用技能列表中是否存在 `superpowers:brainstorming`（superpowers 没有独立的 interview skill，其 brainstorming 内置了"一次一个问题"的访谈式对话流程）：

- **存在** → 通过 Skill 工具调用 `superpowers:brainstorming`，传入用户的任务背景，并声明本次以**澄清模糊点**为目标。同时叠加本方法论的关键约束：**优先提出"回答会改变架构"的问题**。
- **不存在** → 按下方自定义指令执行。

## 自定义指令（fallback）

### 规则

1. **一次只问一个问题**，等用户回答后再问下一个。
2. **优先提出那些"用户的回答会改变架构/技术选型/数据模型"的问题**；纯偏好、可后置的细节往后放或直接跳过。
3. 每个问题给出你倾向的默认答案与理由，让用户可以只回"就按你说的"或纠正你。
4. 结合用户给的背景引导提问，不要问从背景里已能推断的东西。
5. 关键未知基本收敛时**主动停止**，输出一份**结论摘要**：已澄清的决策 + 仍待定的点。

### 边界

- 先不要实现。
- 严格一问一答，不要一次抛出问题清单。

## 原文示例 prompt（供理解意图）

- "就任何含糊之处一次一个问题地访谈我，优先提出那些我的回答会改变架构的问题。"
