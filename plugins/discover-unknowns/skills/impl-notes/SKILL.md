# 实现笔记（Implementation Notes）

按用户给出的规格/计划（随调用传入的参数或当前对话上下文）进入实现，全程维护实现笔记。

## 工作方式

1. 在仓库根目录维护一个临时文件 `implementation-notes.md`（用户更想要 HTML 则用 `.html`）。
2. 按已确认的计划/规格推进实现。
3. 每当撞上一个**迫使你偏离计划的边界情况**：**选择保守的方案**，把「遇到了什么 / 为什么偏离 / 选了哪个方案 / 放弃了什么」记录到笔记的 **`## Deviations`** 小节下，然后**继续推进**，不要停下来反复追问。
4. 同时在笔记里记录关键决策、假设、以及留给下一次尝试的经验。
5. 完成后向用户汇报：做了什么、`Deviations` 里有哪些偏离需要复核。

## 背景（为什么要这样做）

- 无论规划多充分，总有"未知的未知"潜伏——智能体可能在代码里撞上边界情况需要改换思路。笔记让这些发现**可追溯、可复用**。
- 最佳实践：计划满意后**新开会话**，把规划阶段的产物（规格、原型）作为提示词传入——全新上下文窗口 + 全部已编译信息。

## 原文示例 prompt（供理解意图）

- "维护一个 implementation-notes.md 文件。如果你遇到某个边界情况，迫使你偏离计划，就选择保守的方案，把它记录在 'Deviations' 下，然后继续推进。"
